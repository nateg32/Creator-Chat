# Creator Memory V2

This schema is the target foundation for transcript-heavy Creator Bot memory.
It is additive and can run beside the current `scrape_items`, `documents`,
`chunks`, and `embeddings` tables while ingestion is migrated gradually.

## Why V2 Exists

The current schema works, but long creator transcripts need stronger
provenance:

- Raw scraped items should stay separate from normalized knowledge.
- Full transcripts need timestamped segments.
- RAG chunks need stable links back to video/post source material.
- `analysis.md`, `soul.md`, and `runtime_prompt.md` should be versioned.
- Gemini context cache metadata should be per creator and per corpus checksum.
- Retrieval decisions should be logged so answers can be debugged later.

## New Tables

- `creator_platform_accounts`: one creator's connected YouTube, TikTok, Instagram, X, etc. identities.
- `content_documents`: canonical video/post/article records produced from approved scrape items.
- `transcript_segments`: timestamped transcript lines for source-accurate lookup.
- `content_chunks`: RAG-ready chunks with segment and character ranges.
- `content_embeddings`: provider/model-specific vectors for each v2 chunk.
- `persona_analyses`: evidence layer, usually `analysis.md`, generated by Gemini.
- `soul_versions`: versioned `soul.md` files compiled from analyses.
- `runtime_prompts`: active lightweight prompt injected into chat.
- `gemini_context_caches`: Gemini explicit context cache metadata per creator corpus.
- `retrieval_events`: audit trail for vector RAG, Gemini cache lookups, and injected facts.

## Migration Plan

1. Keep the current app reads/writes alive.
2. Dual-write approved scraped content into `content_documents`, `transcript_segments`, and `content_chunks`.
3. Backfill old `documents` and `chunks` into the v2 tables.
4. Write future Gemini analysis output into `persona_analyses`, `soul_versions`, and `runtime_prompts`.
5. Move cache metadata from `creators.gemini_cache_*` into `gemini_context_caches`.
6. Switch RAG reads to v2 tables after parity checks.
7. Retire legacy columns/tables only after production traffic is clean on v2.

## Model Defaults

Gemini is now the default chat provider for this backend:

```env
CHAT_PROVIDER=gemini
GEMINI_CHAT_MODEL=gemini-2.5-flash
GEMINI_ANALYSIS_MODEL=gemini-2.5-pro
GEMINI_CACHE_LOOKUP_MODEL=gemini-3-pro-preview
GEMINI_CACHE_ROUTER_MODEL=gemini-3-flash-preview
```

OpenAI chat remains available:

```env
CHAT_PROVIDER=openai
```
